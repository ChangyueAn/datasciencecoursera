Since the goal is to predict the 20 classes in the testing dataset, so I need to do internal prediction first. After my internal prediction result is good enough, then I can apply the model to the external testing set.

Then, I uploaded different libraries I need and set seed to 12345 so it is reproducible.

library(knitr)

library(caret)

library(rpart)

library(rpart.plot)

library(rattle)

library(randomForest)

library(corrplot)

set.seed(12345)

 

After investigate the training data carefully, I found that there are many columns that contain “NA” value. So I need to do data cleaning before build the model.

So at first,  I used CreateDataPartition to create an internal training dataset and internal testing data set.

Then, in order to clean the dataset. I used nearZeroVar function to return the columns that have almost all “NA” values and store them in NZV. Then, I deleted these columns from both the internal training data and the internal testing data. Then, I tested if a column has more than 95% “NA” value, then deleted the columns too. Finally, since the first 4 variables are identity variables and I don’t need them, so I deleted them.

inTrain<-createDataPartition(pml_training$classe,p=0.7,list=FALSE)

pml_internal_train<-pml_training[inTrain,]

pml_internal_test<-pml_training[-inTrain,]

NZV<-nearZeroVar(pml_internal_train)

pml_internal_train<-pml_internal_train[,-NZV]

pml_internal_test<-pml_internal_test[,-NZV]

pml_testing<-pml_testing[,-NZV]

AllNA<-sapply(pml_internal_train,function(x)mean(is.na(x)))>0.95

pml_internal_train<-pml_internal_train[,AllNA==FALSE]

pml_internal_test<-pml_internal_test[,AllNA==FALSE]

pml_testing<-pml_testing[,AllNA==FALSE]

pml_internal_train<-pml_internal_train[,-c(1:4)]

pml_internal_test<-pml_internal_test[,-c(1:4)]

pml_testing<-pml_testing[,-c(1:4)]

 

Then, I investigated the multicollinearity in the internal training data.

corMatrix_data<-cor(pml_internal_train[,-54])

corrplot(corMatrix_data,order="FPC",method="color",type = "lower",tl.cex = 0.8,tl.col=rgb(0,0,0))

 

Based on the result, we can see that many variables are highly correlated with others. So I need to do PCA analysis to reduce the dimension of the data.

 

pcaObj<-preProcess(pml_internal_train,method = c("center","scale","pca"),thresh = 0.9)

pcaObj

The result shows that: PCA needed 19 components to capture 90 percent of the variance in my internal training data set.

 

Then, I set seed as 12345 and applied the PCA results into my internal training data. Then, I created a control variable called and controlRF and used it in my “random forest” model.

set.seed(12345)

pml_train_pca<-predict(pcaObj,pml_internal_train)

controlRF<-trainControl(method = "cv",number=3,verboseIter = FALSE)

mod_rf<-train(classe~.,data = pml_train_pca,method="rf",trControl=controlRF)

 

Then, I predict the outcome and tested the accuracy.

pml_test_pca<-predict(pcaObj,pml_internal_test)

pred_rf<-predict(mod_rf,pml_test_pca)

confu_pred_rf<-confusionMatrix(pred_rf,pml_test_pca$classe)

confu_pred_rf

The overall accuracy was 97.26%. My “random forest” model did a pretty good job to predict Class A, and Class B. For Class D, it was not as accurate as others.

 

 

Then, I performed tree modeling. Tree model based on the seed 12345.

set.seed(12345)

mod_rpart<-rpart(classe~.,data=pml_train_pca,method="class")

fancyRpartPlot(mod_rpart)

pred_rpart<-predict(mod_rpart,pml_test_pca,type = "class")

confu_pred_rpart<-confusionMatrix(pred_rpart,pml_test_pca$classe)

confu_pred_rpart

plot(confu_pred_rpart$table,col=confu_pred_rpart$byClass,main=paste("Tree - Accuracy =",round(confu_pred_rpart$overall['Accuracy'],4)))

For my tree model, I just got 50.74% overall accuracy, which is pretty bad.

 

Then, I performed GBM model. It was also based on seed 12345.

set.seed(12345)

controlGBM <- trainControl(method = "repeatedcv", number = 5, repeats = 1)

mod_gbm<-train(classe~., data=pml_train_pca,method='gbm',trControl=controlGBM,verbose=FALSE)

pred_gbm<-predict(mod_gbm,pml_test_pca)

confu_pred_gbm<-confusionMatrix(pred_gbm,pml_test_pca$classe)

confu_pred_gbm

plot(confu_pred_gbm$table,col=confu_pred_gbm$byClass,main=paste("GBM - Accuracy =",round(confu_pred_gbm$overall['Accuracy'],4)))

The overall accuracy is 79.98%. It is better than the tree model but it was worse than the random forest model. This model did a pretty good job of predicting Class A.

 

 

Then, in order to try that if I can get even higher accuracy result than random forest. I combine all three models together to predict the internal testing data. After combine all previous three results, I used “random forest” to build the new model.

all_model<-data.frame(pred_rf,pred_rpart,pred_gbm,classe=pml_test_pca$classe)

comFit<-train(classe~.,method="rf",data=all_model)

pred_com<-predict(comFit,pml_test_pca)

con_pred_com<-confusionMatrix(pred_com,pml_test_pca$classe)

con_pred_com

The overall accuracy was 97.26% and it was the same as random forest accuracy. So finally, I decided to use random forest to predict the external dataset “testing”.
